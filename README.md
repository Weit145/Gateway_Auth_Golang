# Система микросервисов Gateway-Auth-Golang

## Обзор проекта

Этот репозиторий содержит интегрированную систему, состоящую из двух взаимосвязанных микросервисов, написанных на Go:
1.  **Сервис аутентификации (`Auth_golang`)**: Отвечает за управление пользователями, регистрацию, аутентификацию, обновление токенов и другие связанные функции.
2.  **Сервис шлюза (`Gateway_golang`)**: Предоставляет HTTP-интерфейс (RESTful API), который проксирует запросы к сервису аутентификации и другим потенциальным внутренним gRPC-сервисам. Он служит точкой входа для внешних клиентов.

Вместе эти сервисы образуют полноценное решение для аутентификации с гибким HTTP-шлюзом.

## Компоненты

### 1. Auth Service (Auth_golang)

*   **Назначение**: Централизованное управление аутентификацией и авторизацией.
*   **Технологии**: Go, gRPC, cleanenv, slog, PostgreSQL (для хранения данных пользователей).
*   **Расположение**: `Auth_golang/`
*   Подробное описание и инструкции по запуску см. в [Auth_golang](https://github.com/Weit145/Auth_golang).

### 2. Gateway Service (Gateway_golang)

*   **Назначение**: HTTP-шлюз, преобразующий HTTP-запросы в gRPC-вызовы к сервису аутентификации и возвращающий HTTP-ответы.
*   **Технологии**: Go, HTTP, gRPC-клиент, cleanenv, slog.
*   **Расположение**: `Gateway_golang/`
*   Подробное описание и инструкции по запуску см. в [Gateway_golang](https://github.com/Weit145/Gateway_golang).

## Используемые технологии (общие)

*   **Go**: Язык программирования для обоих микросервисов.
*   **gRPC**: Для высокопроизводительной межсервисной связи.
*   **cleanenv**: Для удобного управления конфигурацией через YAML-файлы.
*   **slog**: Стандартная библиотека Go для структурированного логирования.
*   **Docker & Docker Compose**: Для контейнеризации и оркестрации сервисов.

## Настройка и запуск с Docker Compose

Система настроена для легкого развертывания с использованием Docker Compose.

### Предварительные требования
*   Установлен Docker.
*   Установлен Docker Compose.

### Запуск всех сервисов

Из корневого каталога проекта выполните:

```bash
docker-compose up --build
```

Эта команда соберет образы Docker для обоих сервисов (Auth и Gateway) и PostgreSQL, а затем запустит их.

*   **Сервис аутентификации**: Будет доступен внутри сети Docker Compose по адресу `auth_service:50051`.
*   **Сервис шлюза**: Будет доступен извне по адресу `http://localhost:8080`.

### Остановка сервисов

Чтобы остановить и удалить контейнеры, сети и тома, созданные `docker-compose up`:

```bash
docker-compose down
```

## Структура каталогов

*   `Auth_golang/`: Исходный код сервиса аутентификации.
*   `Gateway_golang/`: Исходный код сервиса шлюза.
*   `docker-compose.yml`: Конфигурация Docker Compose для оркестрации всех сервисов.
*   `README.md`: Главный файл README для всей системы.
